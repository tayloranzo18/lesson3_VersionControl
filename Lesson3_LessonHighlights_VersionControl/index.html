Highlight #1 What a VCS does

A version control system is a very useful tool in any programmer or designer’s arsenal of file protection. Version control systems record and save changes that a user might make when working on a file, that way they can go back in later and search for a previous version of that specific file for comparison. It means that if something happens on your local storage of the file, you can go back on the Version control system and pull up the last saved version, and that peace of mind is a very good thing to have when working on a major project.

Highlight #2 Remote repositories

A remote repository is a repository that is stored on another server on the web. Remote repositories are very useful in the fact that they allow for multiple people to work and modify the same code at the same time. Remote repositories also can keep files safe because they are on a server and not on a local disk of the user’s. This can be advantageous to a major group project because users can download the file(s) on the remote repository, work on it on their own computers, and send back to the repository their new additions to the file.

Highlight #3 Local repositories

Local repositories on the other hand are files that are stored on a local disk on a singular computer. Local disks are not able to have access to any server directly, and therefore can make working on a group project very difficult since member may have entirely different codes or view of the file since they are not receiving information that comes from a single server. Local repositories can also be difficult and possibly dangerous to have a single local repository because it can easily be lost since there is no system to back it up, and is very difficult to keep track of changes.

Highlight #4 How helpful Git is

Git is a Version Control system that does not require a complicated sever system and can version control items on any desktop through the command line without any complex software. Git was designed for collective projects using a single file, because members can work on the same file on their own computers and then merge the file back to the primary version. Merging different files can be done by adding a command to the system and making two or more different editions into one. Git also can show when and if the different versions hold some kind of conflict with each other.

Highlight #5 Three basic steps

For git, there are three basic steps to the version control system: add, commit, push. When a user is working on a project and decides that they want to update the file, the “add” command is used and the file goes to a staging area. If the user decides that their version is ready to be placed in the repository, they then would “commit” it to the local repository. The final step in this process is when the user will “push” the file back to the remote repository where it can be saved and eventually “pulled” from for editing once again.
